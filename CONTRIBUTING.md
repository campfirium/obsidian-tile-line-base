# Contributing to TileLineBase

Thanks for helping improve TileLineBase.

## Where to Start

- Report bugs with the GitHub bug report template.
- Share feature ideas in GitHub Discussions so people can upvote and compare workflows.
- Ask setup and usage questions in the Q&A discussion category.

## Local Development

Install dependencies:

```bash
npm ci
```

Run the required validation before submitting changes:

```bash
npm run build
```

Use `npm install <package>` only when intentionally adding, removing, or upgrading a dependency.

## Pull Requests

- Keep changes focused on one user-visible behavior or maintenance task.
- Include validation notes in the pull request template.
- Do not edit generated build output directly.
- Keep user-facing strings in the existing localization system.

## Security

Do not open public issues for suspected vulnerabilities. Please follow `SECURITY.md`.
