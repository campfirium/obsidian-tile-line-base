# Security Policy

## Reporting a Vulnerability

Please do not report security issues in public GitHub Issues or Discussions.

Email the maintainer at the contact address listed on the Campfirium GitHub profile, or use a private channel already established with the maintainer. Include:

- A short description of the issue.
- Steps to reproduce.
- The TileLineBase version and Obsidian version.
- Any sample Markdown or files needed to reproduce, with private data removed.

## Scope

TileLineBase processes the active Markdown file where the plugin view is opened. Security reports are especially useful when they involve data loss, unsafe rendering, unexpected file access, dependency risk, or plugin-package integrity.
