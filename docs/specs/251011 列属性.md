# 251011 列属性
## **规范文档 (顶层架构): TileLineBase 列属性 (Column Properties)**

**文档ID:** `SPEC-COLUMN-PROPS-V1.0-ARCH`
**状态:** 已敲定

### **1. 核心理念**

“列属性 (Column Properties)”是 TileLineBase 的核心功能扩展机制。它是一种**声明式的、基于文本的“领域特定语言 (DSL)”**，允许用户通过在“头部配置块”中添加简单的“**特性开关 (Flags)**”和“**键值对 (Key-Value Props)**”，来为特定列“激活”强大的预置功能。

其核心设计哲学是**“功能即属性，组合即应用”**。

### **2. 语法总览**

*   **声明位置:** `tilelinebase` 代码块内。
*   **基本格式:** `列名 (属性1) (属性2: 值) ...`
*   **属性分类 (逻辑层):**
    *   **行为属性 (Behavioral):** 改变列的核心功能和交互方式 (e.g., `(status)`, `(formula: ...)`).
    *   **视觉属性 (Visual):** 改变列的显示样式 (e.g., `(image)`).
    *   **数据属性 (Data):** 影响数据的解析与处理 (e.g., 未来可能的 `(type: number)`)。
*   **国际化:** 所有**属性关键字 (如 `status`, `formula`) 必须是英文**。所有**存储在 Markdown 中的值 (如状态值) 也必须是英文**。UI 层负责将其本地化为用户语言。

### **3. 核心属性列表 (概览)**

| 属性 (Flag/Prop) | 类型 | 核心功能简介 | 状态 |
| :--- | :--- | :--- | :--- |
| **`(status)`** | 布尔特性开关 | 激活**内置的、专业的GTD状态管理系统**。 | **MVP 核心** |
| **`(formula: ...)`** | 键值对 | 将该列定义为**公式列**，支持动态计算。 | **MVP 核心** |
| **`(hide)`** | 布尔特性开关 | 在视图中**默认隐藏**此列。 | **MVP 核心** |
| **`(expandable)`** | 布尔特性开关 | 允许单元格内容**可展开**，用于承载长文本。 | Post-MVP |
| **`(image)`** | 布尔特性开关 | 将单元格内的图片链接，渲染为**缩略图**。 | Post-MVP |
