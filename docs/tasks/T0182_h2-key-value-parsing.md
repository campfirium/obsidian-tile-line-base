# T0182 H2 Key-Value Parsing

## 2025-11-21
- 已创建分支 `feat/T0182-h2-key-value-parsing` 与工作区 `trees/feat-T0182-h2-key-value-parsing`，将工作区 `.git` 指向 `gitdir: ../../.git/worktrees/feat-T0182-h2-key-value-parsing`，并复制根目录 `.vscode/tasks.json` 以沿用 Ctrl+Shift+B 任务。
- 主题：表格解析；类型：缺陷。现象：含中文或英文冒号的句子（如“朋友：‘但是我已经有备忘录了’。”）会被当成多次键值切分，生成多余列。
- 排查假设：键值解析优先匹配全角冒号，可能跳过更早出现的半角冒号，导致对同一行尝试多种切分；应按文本首次出现的冒号类型切分。
- 修复方向：调整 H2 键值解析为“首个全角或半角冒号”，避免重复切分导致列泄漏；补充分割策略的覆盖样例确认句子类文本不会误增列。
- 实施：`resolveColonIndex` 返回全角/半角冒号的最小索引，优先按正文首次冒号拆分；后续针对含多个冒号的句子补充手工样例检查解析结果。
