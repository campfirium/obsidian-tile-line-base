# 图标显示策略（过滤条件删除按钮）

## 背景
- 任务：T0163 看板编辑界面优化 3，需要让过滤条件删除按钮中的垃圾桶图标更清晰。
- 痛点：按钮沿用 `setIcon` 默认尺寸，在暗色主题中显得过小且笔画过细。

## 尝试过的方案（遇到的问题）
1. **整体放大按钮**
   - 作法：把按钮宽高从 28px 直接放大到 `28px * 3`，并同步放大 SVG。
   - 问题：按钮热区、布局被撑开，与周边控件不一致；仍无法保证 SVG 笔画粗细。

2. **仅放大 SVG 宽高（`calc()` 方案）**
   - 作法：保持按钮尺寸，使用 `width: calc(14px * scale)` 改 SVG。
   - 问题：`setIcon` 每次渲染都会重写 `width/height`，主题样式也会覆盖，导致尺寸忽小忽大。

3. **`transform: scale()` 放大内容**
   - 作法：利用 `transform-box: fill-box` + `scale(3)` 只放大图形。
   - 问题：stroke 宽度依旧过细，且在按钮中容易被裁剪或偏移，需要额外补丁，维护成本高。

## 定稿方案
- **回归 Obsidian 推荐写法**：按钮附加 `clickable-icon` 并调用 `setIcon`，继续享受原生 hover/focus、无障碍处理。
- **CSS 变量控制图标尺寸与线宽**：在 `.tlb-filter-view-modal__remove-button` 中定义 `--tlb-filter-remove-icon-size`、`--tlb-filter-remove-icon-stroke`，然后在 `svg` 上使用 `width/height !important` 与 `stroke-width` 指向这些变量，`stroke: currentColor` 保持主题一致。
- 这样按钮热区维持 28px，内部图标可控地放大（默认 20px、线宽 2.2），主题切换不再出现失真。

## 操作指南
1. 创建按钮：`row.createEl('button', { cls: 'clickable-icon tlb-filter-view-modal__remove-button' }); setIcon(button, 'trash-2');`
2. 样式中覆写变量即可调整尺寸：`--tlb-filter-remove-icon-size`、`--tlb-filter-remove-icon-stroke`。
3. 若需在其他位置放大图标，复用相同模式，避免直接修改 `width/height` 或依赖 `transform`。
